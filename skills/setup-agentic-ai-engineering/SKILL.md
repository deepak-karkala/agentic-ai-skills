# Setup: agentic-ai-engineering

One-time initializer for the plugin skill suite. Writes `.agentic/config.yml` with the repo paths that downstream skills need.

Run this once per repo before using the full skill suite. Skills that are purely conversational (architecture design, context engineering, multi-agent topology) work without setup. Skills that need to locate eval assets or write artifacts benefit from or require this config.

## What this skill does

1. Checks whether `.agentic/config.yml` already exists.
2. If it exists, reads the current values and reports which paths are configured and which are absent.
3. If it does not exist, runs the configuration flow below.
4. Writes or updates `.agentic/config.yml`.
5. Offers to add `.agentic/` to `.gitignore` (local config) or confirms it can be committed (shared team config).

## Configuration Flow

Ask each question in order. Accept the default if the user presses Enter. Allow freeform path input.

### Step 1 — Eval assets location (required)

```
Where are your eval configs, traces, and scorecards stored?
Default: evals/
```

This is the only field that `agent-eval-design` requires. Without it, eval audit and regression flows cannot locate assets.

### Step 2 — Design docs location (optional, improves quality)

```
Where are your architecture docs, system design docs, and RFCs stored?
Default: docs/architecture
Press Enter to skip.
```

When set, `agentic-system-design` and `multi-agent-orchestration` read existing docs before generating recommendations. Without this, those skills work from user-provided descriptions only.

### Step 3 — Artifact output location (optional)

```
Where should generated HTML and Markdown artifacts be written?
Default: .agentic/artifacts
Press Enter to use the default.
```

When absent, skills default to `.agentic/artifacts/` and note the path to the user.

### Step 4 — ADR location (optional)

```
Where are your Architecture Decision Records stored?
Default: docs/decisions
Press Enter to skip.
```

### Step 5 — Agent source code location (optional)

```
Where does your agent implementation code live?
Default: src/agents
Press Enter to skip.
```

### Step 6 — Trace logs location (optional)

```
Where are runtime agent traces stored?
Default: logs/traces
Press Enter to skip.
```

### Step 7 — Glossary location (optional)

```
Do you have a project glossary or terminology doc?
Path or press Enter to skip.
```

### Step 8 — Config commit preference

```
Should .agentic/config.yml be committed to version control?
[y] Yes — shared team config, check it in
[n] No — local only, add to .gitignore

Default: y
```

If the user chooses `n`, append `.agentic/` to `.gitignore`. If `.gitignore` does not exist, create it with that entry.

## Output: .agentic/config.yml

Write this file to the repo root after the flow completes:

```yaml
# .agentic/config.yml
# Generated by /agentic-ai-engineering:setup-agentic-ai-engineering
# Version 1 — do not edit field names without running setup again

version: "1"

eval_assets_path: "<confirmed value>"

design_docs_path: "<confirmed value or empty>"
artifact_output_path: "<confirmed value or .agentic/artifacts>"
adr_path: "<confirmed value or empty>"
agent_source_path: "<confirmed value or empty>"
trace_log_path: "<confirmed value or empty>"
glossary_path: "<confirmed value or empty>"
```

Empty string means absent (skill will degrade gracefully). Do not omit keys — always write all fields.

## Post-Setup Report

After writing the file, report:

```
Setup complete. .agentic/config.yml written.

Hard dependency satisfied:
  eval_assets_path: evals/  ✓

Soft dependencies configured:
  design_docs_path: docs/architecture
  artifact_output_path: .agentic/artifacts
  adr_path: (not set — agentic-system-design will work without ADRs)
  agent_source_path: (not set)
  trace_log_path: (not set — agent-eval-design will run in greenfield mode)
  glossary_path: (not set)

Skills that work fully now: all 5 flagship skills
Skills that improve with more config: agentic-system-design (add adr_path), agent-eval-design (add trace_log_path)

To reconfigure, run /agentic-ai-engineering:setup-agentic-ai-engineering again.
```

## Scope Boundaries

This skill does **not**:
- Scaffold source code directories or CI/CD files
- Create eval fixtures, design docs, or traces — it only records where they are
- Force GitHub issue workflow or any project management tooling
- Assume any specific language, framework, or deployment platform
- Run any commands that modify code outside `.agentic/`

## Re-run Behavior

If `.agentic/config.yml` already exists:
1. Show the current values.
2. Ask: "Do you want to update any of these? [y/n]"
3. If yes, re-run the configuration flow with current values as defaults.
4. If no, exit without changes.
